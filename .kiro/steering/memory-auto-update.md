# Living memory — automatic session end

After any session where you **created or changed** code or docs, update memory **before stopping**:

1. Append dated bullets to `docs/ES_MEMORY.md` session log (costing/architecture decisions).
2. Append one row to `docs/SESSION_LOG.md`.
3. Update `docs/LIVE_STATE.md` (phase, next steps).
4. Say: **"Memory updated. [N] files changed."**

Skip for read-only Q&A with no file edits.
